# 用户意图与协作指导

这份文档的目的，是帮助下一位 agent 不只读懂方法，还读懂这个用户。

## 1. 用户当前真正想要的是什么

用户当前真正想要的，不只是：

- 一个六问法 skill
- 一份方法说明
- 一套漂亮的项目结构

更深一层，他想要的是：

- 一个能吸收这些材料的 agent
- 一个能理解他如何和 AI 互动、如何试错、如何修正的人
- 一个能在对话里挖到他背后真实意图的思维教练顾问

## 2. 从 session 里看到的稳定偏好

当前比较稳定的协作偏好包括：

- 偏中文
- 偏朋友 / 私教式沟通，不喜欢太正式
- 偏简单直接，不喜欢过度学术化
- 能接受方法，但不喜欢被流程绑死
- 更愿意先抓目的，再定一个当前具体问题
- 当目的和目标已经比较清楚时，更倾向于先行动
- 很自然地把 AI 当成行动中的搭档，而不只是分析工具

## 3. 用户在方法上的真实张力

用户不是反对六问法本身，而是和下面这些东西有张力：

- 过强的结构化控制感
- 一板一眼地走完整流程
- 在已经够清楚的时候，仍被要求继续深挖
- 把当前具体问题一下子抬成过大的抽象目标

这意味着下一位 agent 需要同时守住两件事：

1. 守住方法边界，尤其是第三问与第四问。
2. 不要把“守边界”做成僵硬的流程压迫。

## 4. 对下一位 agent 的实用指导

如果你要和这个用户继续合作，建议优先这样做：

1. 先区分用户现在是想“看清它”还是“解决它”。
2. 先区分长期方向、当前训练目标、眼前具体问题。
3. 当用户已经抓住目的与目标时，不要强行把他拖回长流程。
4. 如果继续用六问法，第三问只做差距，第四问再做目的校准。
5. 当用户开始想行动时，优先帮他把训练场景、下一步动作和反馈闭环搭起来。

## 5. 对六问法在这个用户身上的更合适用法

对这个用户来说，更合适的理解不是：

`必须标准、完整、按部就班地走完六问法。`

而更像是：

`六问法是一套帮助他看清问题、守住边界、少走弯路的骨架；在骨架不被冲散的前提下，可以更贴近他的行动风格来使用。`

所以较合理的使用策略通常是：

- 前三问守正
- 第四问看背后驱动力
- 如果已经收敛出真实训练目标，就及时切到行动
- 把 AI 当成推进与反馈回路的一部分

## 6. 当前可以长期追踪的主题

下一位 agent 值得持续追踪的主题包括：

- 用户如何更稳定地用 AI 解决自己没做过、不擅长的事情
- 六问法对他来说怎样用才不僵
- 服务模式与觉察模式怎样切换才最自然
- 怎样更好地利用老师 / 私教资源来安排训练

## 7. 最容易踩的坑

最容易踩的坑有这些：

1. 一上来继续围着 `skills/yc-think` 打转。
2. 把所有问题都拉回完整六问法流程。
3. 在用户已经准备行动时，还强行深挖第五问。
4. 把当前问题提得过大，变成空泛的人生方向。
5. 只看结论，不读原始 session，结果看不见用户真实的语言与思路变化。
