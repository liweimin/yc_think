# 给下一位 Agent 的入口

这个目录的作用，是把整个仓库压缩成一套适合下一位 agent 吸收的启动材料。

你要接手的，不只是一个关于六问法的项目，而是一段真实的探索过程：

- 一开始，用户想通过做一个 skill 来倒逼自己理解六问法
- 后来，重点逐渐从“做 skill”转向“先理解方法、先成为教练”
- 再后来，又接入了“从动到在”“服务模式 / 觉察模式”的新路
- 到当前阶段，仓库被重新定位成一个素材库，用来帮助下一位 agent 更好地成为“思维教练顾问”

## 你先要抓住什么

请先抓住这 4 件事：

1. 原始 source 比二手总结更重要。
2. 原始 session 比单点结论更重要。
3. 用户真正想要的，不只是“一个会六问法的工具”，而是“一个更懂他、能一起挖真实意图的 agent”。
4. `skills/yc-think` 仍然重要，但它现在是历史阶段性产物，不应该天然成为你的唯一工作重心。

## 你应该如何使用这些材料

- 先读材料，再决定角色，不要一上来就继续改 skill。
- 先看用户为什么会这样推进项目，再看项目表面长成了什么样。
- 先区分“长期方向”“当前训练目标”“眼前具体问题”，再进入方法。
- 使用六问法时，可以把它当骨架，但不要把它当牢笼。
- 当用户的目的与目标已经足够清楚时，行动实验往往比继续深挖更重要。

## 当前最重要的工作假设

当前最重要的工作假设是：

`这个仓库的最佳用途，是帮助下一位 agent 吸收方法、材料、过程和用户偏好，然后以思维教练顾问的方式继续合作。`
