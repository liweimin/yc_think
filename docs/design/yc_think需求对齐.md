# yc_think 需求对齐

## 1. 当前结论

`yc_think` 第一版定位已经基本明确：

- 名称固定为 `yc_think`
- 它是一个由用户显式调用的元 skill
- 它的核心角色是“教练”，不是单纯代做工具
- 它的主方法骨架是“界定问题六问法”
- 它的目标不仅是帮用户把问题分析清楚，还要帮助用户逐步内化这种思考方式

一句话定义：

**`yc_think` 是一个由用户显式调用的教练型元 skill，用六问法作为主骨架，在结构化问题界定的同时，训练用户形成更清晰、更有逻辑的问题分析习惯。**

## 2. 已对齐的关键决策

### 2.1 skill 名称

- 固定名称：`yc_think`

### 2.2 触发方式

- 不依赖模糊语义自动触发
- 由用户明确说要使用 `yc_think`

这意味着它属于“显式调用型元 skill”。

### 2.3 第一版输出倾向

已确认采用：

- `B. 结果优先，解释适中`

含义：

- 不会写成长篇教程
- 也不会只给结论
- 会在关键步骤解释“为什么这样做”

### 2.4 第一版专业框架支持

已确认采用：

- `B. 要，但先做轻量版`

含义：

- skill 要具备识别专业领域并调用经典框架的能力
- 但第一版先做轻量级，不追求全行业覆盖

### 2.5 用户回答弱时的引导方式

已确认采用：

- `A. 可以，而且应该这样做`

含义：

- 当用户答不上来时，skill 不只是追问
- 要主动给出“回答方向”“思考提示”“候选结构”
- 这样可以显著降低用户的认知负担和情绪负担

### 2.6 第一版交互形态

已确认采用：

- `A. 先把单轮做好`

含义：

- 第一版优先保证一次使用就能稳定完成完整分析
- 多轮长期记忆感不是第一版重点

## 3. skill 的核心定位

`yc_think` 不是为了替用户直接给答案，而是为了做 3 件事：

1. 帮用户识别“自己提的问题是否真的是问题”
2. 帮用户把问题一步步界定清楚
3. 在过程中教会用户一种更稳定的思考方式

因此，它必须同时承担两种职责：

### 3.1 分析职责

- 判断问题类型
- 判断是否适合六问法
- 跑完整六问法
- 在需要时重开新一轮六问法
- 输出最终真正问题

### 3.2 教练职责

- 解释为什么当前这一步要这么做
- 提供合理的回答方向
- 用逻辑结构帮助用户组织信息
- 在用户卡住时降低思考门槛
- 让用户逐渐学会这套方法

## 4. 适用边界

这是你新补充的最关键要求之一，必须作为 skill 的正式前置条件。

### 4.1 六问法最适合的问题类型

`yc_think` 默认优先处理：

- 差距不明确的行动类问题

这类问题通常具备这些特征：

- 需要行动主体去解决
- 用户表达的问题比较模糊、抽象或混杂
- 目标、现状、差距至少有一项不清楚
- 用户可能把“解决方案”误当成“问题”

### 4.2 使用前必须先做问题判断

在正式进入六问法前，`yc_think` 必须先判断：

- 这是不是一个适合六问法的问题

这一步是前置闸门，不可跳过。

## 5. 前置判断机制

`yc_think` 的第一步不应该直接开始六问法，而应该先做“问题适配判断”。

### 5.1 前置判断的目标

判断用户当前输入属于哪一类：

1. 适合直接进入六问法
2. 不适合六问法，但可以简单分类并给出更直接建议

### 5.2 第一版推荐的问题分类

为了兼顾实用性和稳定性，第一版建议采用下列分类：

#### A. 差距不明确的行动类问题

处理方式：

- 进入完整六问法

典型例子：

- 我想提升团队效率，但不知道真正问题在哪
- 我总拖延学习任务，怎么办
- 我觉得这件事做不下去，但说不清卡在哪

#### B. 目标不清类问题

处理方式：

- 不直接进入完整六问法
- 先帮助用户明确目标

典型例子：

- 我想变得更厉害
- 我想更有前途

#### C. 现状不清类问题

处理方式：

- 先帮助用户澄清事实、现象、数据、样本

典型例子：

- 我感觉团队状态不太对
- 我感觉网站不太行

#### D. 纯事实查询 / 信息查询问题

处理方式：

- 不用六问法
- 直接用普通问答或查询式处理

典型例子：

- 某个方法怎么用
- 某个指标是什么意思

#### E. 方案选择类问题

处理方式：

- 先做轻量决策分析
- 如果背后仍是差距不明确的行动问题，再切回六问法

典型例子：

- 我要不要离职
- 我要不要换岗位

#### F. 情绪表达 / 倾诉类问题

处理方式：

- 先接住情绪并澄清诉求
- 如果用户希望继续分析，再转入问题界定

典型例子：

- 我现在很烦，不知道怎么办

### 5.3 第一版前置判断输出

如果适合六问法：

- 说明为什么适合
- 进入六问法

如果不适合六问法：

- 说明这属于什么问题
- 说明为什么不适合直接用六问法
- 给出建议的处理方式

这一步也有教育意义：

- 帮助用户逐步形成问题分类意识

## 6. 教练风格要求

这是 `yc_think` 和普通六问法执行器最大的区别。

### 6.1 基本风格

第一版默认采用：

- 温和教练型
- 降低压力
- 降低作答门槛
- 解释适中

### 6.2 教练不是考试官

skill 不应表现为：

- 连续丢问题给用户自己扛
- 要求用户一次给出完美回答
- 用生硬方式指出用户表达不清

skill 应表现为：

- 帮用户整理已有表达
- 解释这一步为什么要问
- 给用户几个可参考的回答方向
- 在必要时帮用户把模糊表达翻译成可分析对象

### 6.3 当用户卡住时的默认动作

优先顺序建议如下：

1. 先解释这一步的目的
2. 再给 2-4 个回答方向
3. 再给一个简短示例
4. 最后请用户从中选择或补充

## 7. 回答方向与逻辑引导

你特别强调了，第一版要主动提供回答方向，以减轻用户作答难度和情绪压力。这一点应写成硬要求。

### 7.1 skill 必须提供回答方向

在以下情况，`yc_think` 应主动给出回答方向：

- 用户回答很弱
- 用户表达混乱
- 用户明显不知道从哪里开始
- 当前问题适合用固定框架辅助

### 7.2 优先使用的三类逻辑结构

这是第一版的核心逻辑支撑：

- 结构
- 顺序
- 重要性

### 7.3 三类逻辑结构的典型用途

#### 结构

适合用于：

- 拆组成部分
- 拆角色
- 拆模块
- 拆因素类别

示例：

- 人 / 机 / 料 / 法 / 环
- 管理者 / 执行者
- 个人 / 团队 / 外部环境

#### 顺序

适合用于：

- 拆流程
- 拆阶段
- 拆时间前后关系

示例：

- 事前 / 事中 / 事后
- 获取线索 / 转化 / 交付 / 复盘
- 输入 / 处理 / 输出

#### 重要性

适合用于：

- 做优先级
- 找关键矛盾
- 做原因排序

示例：

- 影响最大 / 次大 / 最小
- 高频 / 中频 / 低频
- 关键原因 / 次要原因 / 噪声原因

## 8. 框架使用策略

这是 `yc_think` 的重要增强层。

### 8.1 框架使用优先级

第一版建议固定为：

1. 优先使用专业领域经典框架
2. 其次使用通用经典逻辑框架
3. 最后使用“先发散、再收敛成框架”

### 8.2 适合优先调用的经典框架

第一版建议优先支持这些类型：

- 通用问题分析框架
- 流程分析框架
- 管理与执行分工框架
- 人机料法环
- 目标/现状/差距类框架

如果识别到明显专业领域，可尝试调用：

- 产品
- 项目管理
- 销售
- 运营
- 团队管理

但第一版不追求覆盖全面。

### 8.3 没有框架时的处理

如果没有合适框架，skill 应：

1. 先引导用户发散
2. 再帮助用户分组
3. 再帮助用户命名和收敛

也就是：

- 先把内容摊开
- 再整理成结构

## 9. 六问法执行要求

第一版必须保留完整六问法骨架。

### 9.1 核心流程

1. 判断是否适合六问法
2. 如果不适合，先分类并给建议
3. 如果适合，进入六问法
4. 第 1 问：目标是什么
5. 第 2 问：现状是什么
6. 第 3 问：差距是什么
7. 第 4 问：目标背后的目的是什么
8. 第 5 问：导致现状的原因是什么
9. 第 6 问：差距真正的原因是什么
10. 重新界定真正的问题

### 9.2 关键硬规则

- 第 4 问必须使用 `0.5Why`
- 第 5 问必须使用 `0.5Why`
- 第 3 问或第 4 问发现原问题失效时，必须重开
- 第 5 问要优先借助框架

### 9.3 重开机制

如果在以下阶段发现问题需要重构：

- 第 3 问：发现原始问题并不是问题
- 第 4 问：发现原目标只是初步解决方案

则必须：

1. 说明为什么当前问题失效
2. 生成新的问题
3. 基于新问题重开六问法

## 10. 输出要求

第一版输出应尽量贴近你提供的六问法示例结果，同时增加适量说明。

### 10.1 输出的三层结构

建议固定输出为：

#### A. 正式分析结果

尽量贴近六问法示例，包括：

- 原始问题
- 六问法过程
- 必要的重开过程
- 最终真正问题
- 限制因素
- 可改变因素
- 对策方向

#### B. 教练说明

简要说明：

- 这次为什么这样界定
- 哪些地方是关键判断
- 下次用户自己可以怎么做

#### C. 补充说明

可选补充：

- 当前分析的置信度
- 还缺什么事实
- 如果继续深化，下一步该做什么

### 10.2 输出风格

- 结构清楚
- 结果优先
- 解释适中
- 不写成大段说教

## 11. 第一版不追求的内容

为了保证第一版收敛，不建议一开始就追求：

- 自动触发
- 全行业专业框架库
- 长期多轮记忆式陪跑
- 超复杂评估脚本
- 一开始就覆盖所有问题类型

## 12. 当前已对齐的第一版定义

综合以上内容，第一版 `yc_think` 可以定义为：

**一个由用户显式调用的教练型元 skill。它先判断当前输入是否属于适合六问法处理的“差距不明确的行动类问题”；若不是，则先帮助用户识别问题类型并给出更合适的处理建议；若是，则使用界定问题六问法作为主流程，并在执行过程中主动提供回答方向、逻辑结构和必要解释，帮助用户在完成问题界定的同时，逐步内化结构化思考习惯。**
