# yc_think 手工测试用例

## 目标

这份清单用于人工试跑第一版 `yc_think`。

建议先测：

1. 一个简单 fit 问题
2. 一个中等 fit 问题
3. 一个复杂 fit 问题
4. 一个 non-fit 信息问题
5. 一个 non-fit 情绪问题

## 1. 简单 fit

### Prompt

Use yc_think. 我总是拖延本月学习任务，但我也说不清真正的问题是什么。请按这个方法帮我分析。

### 观察点

- 是否判断为适合六问法
- 是否完整走到六问法
- 第 4 问和第 5 问是否显式出现 0.5Why
- 是否把问题拆到“认知层”和“执行层”
- 最终问题是否比原始问题更具体

## 2. 中等 fit

### Prompt

Use yc_think. 我想在两个月内提升团队工作效率，但现在只是感觉流程低效，真正问题是什么我不清楚。请你像教练一样带我梳理。

### 观察点

- 是否判断为适合六问法
- 是否补“专业支撑”或团队/流程框架
- 是否主动提供回答方向
- 是否输出限制因素和可变因素
- 最终问题是否接近“六问法示例”的结构

## 3. 复杂 fit

### Prompt

Use yc_think. 我想这个月离职，因为我觉得在公司没有前途了。你不要直接给建议，先帮我分析这是不是我真正的问题。

### 观察点

- 是否识别出这可能是方案化问题或选择问题
- 是否在第 4 问附近触发重开
- 是否保留原始问题和重开原因
- 是否能产出新的真正问题

## 4. non-fit 信息问题

### Prompt

Use yc_think. SMART 原则是什么意思？

### 观察点

- 是否判断为不适合六问法
- 是否给出问题分类
- 是否直接回答，不硬套六问法

## 5. non-fit 情绪问题

### Prompt

Use yc_think. 我现在非常烦，感觉什么都不想做。

### 观察点

- 是否先接住情绪
- 是否判断这不是立即进入六问法的最佳时机
- 是否询问用户是想分析、支持，还是两者都要

## 6. 体验反馈建议

试跑时建议额外记录：

- 用户是否感到被追问得太累
- 回答方向是否足够具体
- 解释是否太多或太少
- 输出是否真正有助于下一步行动
