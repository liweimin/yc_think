# yc_think v0.1 手工测试用例

## 这版重点看什么

和上一版不同，这一版优先测试：

1. 是否默认互动
2. 是否默认中文
3. 是否会停下来等用户回答
4. 是否会主动给回答方向
5. 是否只在明确要求时进入草案模式

## 1. 简单 fit，默认互动

### Prompt

Use yc_think. 我总是拖延本月学习任务，但我也说不清真正的问题是什么。请按这个方法帮我分析。

### 期望

- 判断为适合六问法
- 用中文输出
- 不直接把整套六问法都写完
- 先说明第一步要做什么
- 给 2-4 个回答方向
- 提一个具体问题并停下来等用户

## 2. 中等 fit，默认互动

### Prompt

Use yc_think. 我想在两个月内提升团队工作效率，但现在只是感觉流程低效，真正问题是什么我不清楚。请你像教练一样带我梳理。

### 期望

- 判断为适合六问法
- 使用中文
- 补团队/流程视角
- 主动给回答方向
- 默认分步推进，不直接写完整终局分析

## 3. 复杂 fit，可能触发重开

### Prompt

Use yc_think. 我想这个月离职，因为我觉得在公司没有前途了。你不要直接给建议，先帮我分析这是不是我真正的问题。

### 期望

- 先分类，不急着给建议
- 能识别这可能是“方案化问题”或“选择问题”
- 若进入六问法，第 4 问附近应可能重开
- 仍然保持互动，不直接一口气给完整答案

## 4. 非 fit 信息问题

### Prompt

Use yc_think. SMART 原则是什么意思？

### 期望

- 判断为不适合六问法
- 说明这是什么问题
- 直接回答，不硬套六问法

## 5. 非 fit 情绪问题

### Prompt

Use yc_think. 我现在特别烦，什么都不想做。

### 期望

- 先接住情绪
- 不强推六问法
- 问用户更想要分析、支持，还是两者都要

## 6. 草案模式测试

### Prompt

Use yc_think. 先别互动，直接按现有信息给我一版初步分析，后面我再修正。

### 期望

- 明确说明这是“初步草案”
- 标明假设
- 使用六问法结构
- 最后邀请用户修正
