# yc_think v0.2 手工测试用例

## 这版重点看什么

这版重点不再是“是否互动”，而是：

1. 第 1-3 问是否默认先问
2. 回答方向是否只在需要时才出现
3. 第 3 问是否不再给人“提前 0.5Why”的感觉

## 1. 简单 fit，先问不先提示

### Prompt

Use yc_think. 我总是拖延本月学习任务，但我也说不清真正的问题是什么。请按这个方法帮我分析。

### 期望

- 判断为适合六问法
- 默认中文
- 先问当前步骤
- 不要在用户还没回答前就默认给一堆回答方向
- 如果用户回答后仍然模糊，再补提示

## 2. 用户主动要提示

### Prompt

Use yc_think. 我总是拖延本月学习任务，但我不知道第1问该怎么答。你给我几个思路。

### 期望

- 可以给方向
- 方向数量控制在 2-4 个
- 方向要服务当前步骤，不要提前跳到后面

## 3. 第3问不提前出现 0.5Why 体感

### Prompt

Use yc_think. 我想在两个月内提升团队工作效率，但现在只是感觉流程低效，真正问题是什么我不清楚。请你像教练一样带我梳理。

### 期望

- 第 3 问主要是确认差距
- 可以重开
- 但不要显式使用 `0.5Why`
- 也不要用太强的“是不是这句”验证句式反复逼问

## 4. 第4问允许显式 0.5Why

### Prompt

Use yc_think. 我想这个月离职，因为我觉得在公司没有前途了。你不要直接给建议，先帮我分析这是不是我真正的问题。

### 期望

- 到第 4 问时可以显式出现 `0.5Why`
- 这时用户不会觉得用错地方

## 5. 草案模式仍可用

### Prompt

Use yc_think. 先别互动，直接按现有信息给我一版初步分析，后面我再修正。

### 期望

- 仍能进入草案模式
- 但必须标明是草案
- 不影响默认互动模式
