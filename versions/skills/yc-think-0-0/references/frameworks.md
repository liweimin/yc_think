# Frameworks

## Priority order

Use frameworks in this order:

1. domain-specific framework
2. general logic framework
3. brainstorm then cluster

The goal is not to sound smart. The goal is to avoid blind spots.

## General logic frameworks

Use these when no stronger domain framework is obvious.

### Structure

Use for:

- components
- actors
- factor categories
- system parts

Examples:

- people / process / tool / environment
- manager / executor
- personal / team / external
- input / process / output

### Order

Use for:

- time sequence
- process steps
- dependency order

Examples:

- before / during / after
- upstream / core step / downstream
- planning / execution / review

### Importance

Use for:

- prioritization
- ranking causes
- highlighting key leverage points

Examples:

- highest impact / medium / low
- critical / supporting / minor
- frequent / medium / rare

## Light domain frameworks

Use these only when the domain is reasonably clear.

### Product or website

- traffic
- conversion
- content
- experience
- performance
- measurement

### Project or workflow

- role clarity
- process design
- handoff points
- tooling
- feedback loop
- review mechanism

### Team management

- expectations
- capabilities
- incentives
- coordination
- supervision
- communication

### Sales or operations

- volume
- conversion
- cycle time
- quality
- repeatability
- control points

## If no framework fits

Use this fallback:

1. ask the user to list everything that may matter
2. cluster similar items
3. name each cluster
4. order clusters by structure, order, or importance
5. continue with analysis
