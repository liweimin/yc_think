# Output Format

Use this structure by default.

## 1. Fit judgment

- six-question fit: yes or no
- problem type:
- why:
- next action:

If the answer is no, stop after giving the simpler handling path.

## 2. Original problem

- original statement:

## 3. Support layer

- likely domain:
- framework used:
- why this framework:

## 4. Round-by-round process

For each round, include:

### Round N

- question 1 goal:
- question 2 current state:
- question 3 gap:
- question 4 purpose behind the goal:
- question 4 0.5Why checks:
- question 5 causes behind the current state:
- question 5 0.5Why checks:
- question 6 true cause of the gap:

### Restart decision

- restart needed: yes or no
- if yes, why:
- new problem:

## 5. Final result

- final real problem:
- constraints:
- changeable factors:
- first-pass action directions:

## 6. Coaching notes

Keep this short.

Include:

- why the key judgment mattered
- what the user can reuse next time
- what is still uncertain

## Style notes

- keep the structure explicit
- keep explanations short
- do not collapse the process into a single summary
- the final result should resemble a worked six-question example, not a vague essay
